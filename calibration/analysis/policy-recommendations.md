# Policy recommendations

Each row: current value → proposed value, the data point that motivates the change, and the risk of the change. Confidence tag uses the guide's protocol: `Established` (clear), `Inferred` (correct given rules; reviewer would not flag), `Uncertain`.

Evidence base:
- 8-repo 50-commit-window matrix (`calibration/findings/<repo>.json`) — 45 errors, 141 warnings.
- 5–7 merged-PR baselines per repo where the rate-limited window allowed (`calibration/findings/prs/`) — humans approved each, so any error is a high-confidence FP.

## R-1 — Add `excluded_path_globs` policy field (NEW)

**Current:** no field. `is_excluded_path()` only consults the hardcoded `EXCLUDE_DIRS` set.

**Proposed:** add policy field with default globs:

```json
"excluded_path_globs": [
  "**/migrations/**",
  "**/generated/**", "**/__generated__/**", "**/_generated/**",
  "**/*.pb.go", "**/*.pb.cc", "**/*.pb.h", "**/*.pb.py",
  "**/*_pb2.py", "**/*_pb2_grpc.py",
  "**/*.generated.*",
  "**/dist-cjs/**", "**/dist-es/**", "**/dist-types/**",
  "clients/*/src/commands/**", "clients/*/src/models/**",
  "clients/*/src/schemas/**", "clients/*/src/protocols/**",
  "**/admin/static/admin/**", "**/static/admin/**",
  "docs_src/**",
  "**/python_version.py"
]
```

**Wire into:** `is_excluded_path()` should test each glob via `fnmatch` and short-circuit `is_source_path()`.

**Data motivating this:**
- aws-sdk-js 50-commit window: **18 bloat errors + 91 bloat warnings**, ~100% of which are in `clients/*/src/{commands,models,schemas}/*.ts` (auto-generated from AWS Smithy).
- pydantic 50-commit window: 1 bloat error on `pydantic-core/src/self_schema.py +6852 lines` — file header literally says "auto-generated by generate_self_schema.py, DO NOT edit manually".
- typescript 50-commit window: 2 bloat errors on `*.generated.d.ts` files.
- grpc 50-commit window: 14-way duplicate-block hit on `**/python_version.py` (intentional per-package version constants).
- django merged-PR pr-21136 (Biome migration): bloat on `django/contrib/admin/static/admin/js/urlify.js +363` — vendored frontend asset.
- fastapi merged-PR pr-15280 (`@app.vibe()`): quality escape at `docs_src/vibe/tutorial001_py310.py:12` — pedagogical sample code.

**Risk:** legitimate hand-authored code happens to live in `**/generated/**` somewhere. Mitigation: globs are policy-overridable per-repo. The defaults only zero out canonical generator-output paths plus three repo-specific patterns we observed (admin/static, clients/*/src/commands, docs_src).

**Confidence: Established.** 100+ FP findings across 4 repos in the data set, all in well-known generator paths.

## R-2 — Add `framework_override_names` allowlist (NEW)

**Current:** no field. `same_behavior_name()` returns score 100/90 for any pair where names or token-tuples are identical.

**Proposed:** add list of method/function names that frameworks REQUIRE to be re-defined across consumers. When both symbols in a candidate reuse pair match an entry, suppress the finding (or demote to `score: 25`). Defaults include:

- Django ORM/admin/forms: `validate`, `clean`, `save`, `save_model`, `save_formset`, `get_queryset`, `get_form`, `as_sql`, `as_view`.
- Django REST Framework: `validate`, `to_internal_value`, `to_representation`, `get_serializer`.
- Python descriptor/iterator/operator dunders: `__init__`, `__call__`, `__enter__`, `__exit__`, `__next__`, `__set__`, `__get__`, `__hash__`, `__eq__`, etc.
- React/Angular lifecycle methods: `componentDidMount`, `render`, `ngOnInit`, etc.

**Data:**
- sentry pr-window: `validate` flagged across `serializers.py` files — DRF-mandated method name.
- sentry pr-window: `__set__` flagged against `set` — Python descriptor protocol.
- pydantic pr-window: `__next__` (Rust pyo3) flagged against `next` — iterator protocol.
- django pr-window: `as_sql` flagged across two SQL-emitting modules — Django QuerySet API.

**Risk:** allowlist prevents catching genuine re-implementations of these names. Mitigation: framework-mandated names are by definition repeated by design; the gate's job is not to flag them. The reuse detector retains its score-100 path for non-framework collisions.

**Confidence: Established.** 5+ FPs in this exact class across 3 repos.

## R-3 — Suppress private/public sibling pairs in reuse detector (NEW BEHAVIOR + FLAG)

**Current:** `_save_formset` and `save_formset` both tokenize to `("save", "formset")`. `same_behavior_name()` returns 90 because tokens are identical.

**Proposed:** add policy flag `reuse_suppress_private_public_siblings: true`. When two symbols' raw names differ only by leading-underscore prefix(es) (e.g., `_foo`/`foo`, `__foo`/`foo`, `__foo__`/`foo`), demote score by 70 (so the previous 90 lands at 20, below the warning threshold of 45).

**Data:**
- django: `_save_formset`/`save_formset`, `_walk_items`/`walk_items` (same file private/public pair), `as_sql`/`_as_sql`.
- sentry: `_get_cache_key`/`get_cache_key`, `__set__`/`set`.
- pydantic: `__next__`/`next`.

**Risk:** misses an actual case where someone duplicates a public function and merely prefixes with `_`. Mitigation: if the underscore-stripped names match AND the bodies are similar (token overlap > 0.66), the abstraction-sniff and duplicate-added-block detectors still fire on the body — defense in depth.

**Confidence: Established.** 8+ FPs of exactly this shape across 3 repos.

## R-4 — `reuse_min_duplicate_count: 3` (NEW field)

**Current:** the duplicate-added-block detector fires on N≥2 occurrences.

**Proposed:** add `reuse_min_duplicate_count: 3`. Skip duplicate findings where N=2 (most common 2-instance hit is two-place-coincidence, not copy-paste).

**Data:**
- django pr-21152: 1 duplicate hit, count=2, between `models/fields/__init__.py` and `models/fields/reverse_related.py`. Reviewers shipped this PR; the duplicate was either already there or trivially small. Flagging at N=2 is too eager.
- aws-sdk-js: 413 duplicates total — countless N≥3 hits on the SDK template repetition. R-1 (`excluded_path_globs`) handles these; R-4 separately handles the N=2 case.

**Risk:** may miss a genuine 2-place copy-paste. Mitigation: at N=2 with high token overlap and risky-block tokens, the existing risky-block reuse detector (`risky_added_blocks`) still fires through a different path.

**Confidence: Inferred.** Single direct FP observation; further FPs likely.

## R-5 — `bloat_large_file_lines: 1200 → 1500`

**Current:** 1200 lines triggers "large source file" classification.

**Proposed:** raise to 1500.

**Data:**
- django: `query.py` is 2892 lines, `query.py` 3024, `models/fields/__init__.py` 2972, `admin/options.py` 2632. These are all on `main` for a reason — Django's mature, stable monoliths. Even 1500 still catches them as "large", but reduces warning noise from the 1200–1500 tier.
- pydantic: largest hand-authored file under 1200 except `_generate_schema.py` 2922. 1500 unchanged here.
- typescript: many files in `src/lib/` are over 1500 (`dom.generated.d.ts` 18000+); all generated, handled by R-1.
- aws-sdk-js: `clients/client-batch/src/models/models_0.ts` 9973 lines (generated, handled by R-1).
- nextjs: `packages/next/src/server/app-render/app-render.tsx` is large; counts are in warning tier.

**Risk:** legitimate "this file is too large, please split" warnings get suppressed in the 1200–1500 band. Mitigation: this is a warning-tier change; errors still fire on growth-of-large + exceeding `bloat_total_added_error_lines`.

**Confidence: Inferred.** Reasonable evidence from Django; less direct from others.

## R-6 — `max_design_markers: 2 → 4` and add `max_design_marker_density_per_100_lines: 6.0` (NEW)

**Current:** `max_design_markers: 2` — fires when 2+ markers in a single contract.

**Proposed:**
- raise raw threshold to 4 (small files retain the original protection),
- add density threshold: skip the finding if the file has ≥100 lines and density is below 6.0 markers per 100 lines.

**Data:**
- pydantic `_internal/_generate_schema.py` (2922 lines): manual count of `Protocol|TypeVar|Generic\[|Abstract|Base` ≈ 40+ matches — density ≈ 1.4/100. Above 2 raw, well below density 6.0. Should NOT fire.
- fastapi `routing.py`, `dependencies/utils.py`: typing-heavy by design. Density typically <5/100.
- A small handcrafted file with 4 markers in 50 lines = density 8.0/100 — STILL fires, which is the intent (gratuitous abstraction in small surface).

**Risk:** small overdesigned files in the 2–3 marker range now slip through. Mitigation: in practice 2-marker files are usually `Generic[T]` + `TypeVar` for one well-justified type alias.

**Confidence: Inferred.** Manual marker counts on pydantic/fastapi files; abstraction-sniff path was not exercised by `check`-only runs.

## R-7 — Cleanliness baseline confirmed (no policy change)

**Current:** test `test_gate_check_creates_no_repo_artifacts` passes locally.

**Data:** `calibration/findings/cleanliness.log` shows `clean: <repo>` for all 8 repos, including the 600-second TypeScript run.

**Confidence: Established.** No change required, but worth pinning with a calibration test (Phase 5 / PR-3).
